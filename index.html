🖥️ TechStore API

REST API untuk Sistem Kasir Toko HP dan Laptop berbasis Node.js dan MySQL.

🚀 Pendahuluan

Sistem manajemen kasir konvensional seringkali menghadapi kendala dalam hal sinkronisasi data ketersediaan barang secara real-time, pencatatan transaksi yang rentan human-error, hingga lambatnya pembuatan rekapitulasi laporan bisnis harian. Hal ini menuntut adanya digitalisasi melalui sebuah arsitektur backend tersentralisasi yang tangguh dan aman.

Tujuan Utama Project: Pembuatan TechStore API ini bertujuan untuk memecahkan masalah tersebut dengan merancang sistem RESTful API cerdas yang menjembatani antarmuka kasir (*Frontend*) dengan sentra basis data (*Database*). API ini dibuat untuk mengotomatisasi beberapa proses vital bisnis seperti:

  • Melakukan sentralisasi inventaris (stok hp/laptop) dan mencegah penjualan barang yang stoknya sudah habis.
  • Mengotomatisasi pemotongan sisa stok di gudang secara mutlak setiap kali keranjang transaksi *checkout* berhasil diproses.
  • Menerapkan keamanan *layer* ganda melalui autentikasi **API Key** sehingga data toko tidak bocor kepada pihak eksternal.
  • Mengagregasikan jutaan data transaksi menjadi *Business Intelligence (Laporan Cerdas)*, seperti mengetahui produk apa yang paling laris, pelanggan yang paling sering belanja, hingga peringatan sisa stok menipis bagi pemilik toko.

📌 Judul Project & Studi Kasus

Judul: TechStore API - Point of Sales Backend

Studi Kasus: Pengembangan arsitektur backend Point of Sales (Sistem Kasir) untuk sebuah toko ritel yang menjual HP dan Laptop. Sistem ini dirancang untuk mempermudah kasir dalam melakukan manajemen inventaris (Products), mendata informasi pembeli (Customers), memproses checkout belanjaan secara *real-time* yang langsung memotong sisa stok produk (Transactions), dan memberikan visibilitas kepada pemilik toko melalui rekapitulasi data penjualan cerdas (Laporan Laba & Sisa Stok).

🚀 Cara Menjalankan Project

1. Eksekusi File di Terminal

Buka Terminal (atau Command Prompt / Git Bash), lalu ikuti perintah berikut secara berurutan:


  1. Arahkan ke folder project: cd TechStoreAPI
  2. Install dependencies: npm install

2. Buat Database

Buka phpMyAdmin di http://localhost/phpmyadmin dan jalankan SQL berikut:

CREATE DATABASE techstore;

USE techstore;

CREATE TABLE products (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nama VARCHAR(100) NOT NULL,
    kategori VARCHAR(50) NOT NULL,
    harga INT NOT NULL,
    stok INT NOT NULL
);

CREATE TABLE customers (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nama VARCHAR(100) NOT NULL,
    telepon VARCHAR(20) NOT NULL,
    alamat TEXT NOT NULL
);

CREATE TABLE transactions (
    id INT AUTO_INCREMENT PRIMARY KEY,
    customer_id INT NOT NULL,
    total_harga INT NOT NULL,
    tanggal DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (customer_id) REFERENCES customers(id)
);

CREATE TABLE transaction_items (
    id INT AUTO_INCREMENT PRIMARY KEY,
    transaction_id INT NOT NULL,
    product_id INT NOT NULL,
    nama VARCHAR(100) NOT NULL,
    harga INT NOT NULL,
    jumlah INT NOT NULL,
    subtotal INT NOT NULL,
    FOREIGN KEY (transaction_id) REFERENCES transactions(id),
    FOREIGN KEY (product_id) REFERENCES products(id)
);

3. Jalankan Server

Kembali ke terminal yang tadi (pastikan masih di dalam folder TechStoreAPI), lalu jalankan:

node server.js

Jika berhasil, akan muncul tulisan Server berjalan di http://localhost:3000

⚠️ Catatan Penting Penggunaan :id

Penjelasan Path Variables

Beberapa endpoint menggunakan :id di URL. Ganti :id dengan angka ID yang sesuai.


Contoh (Lengkap dengan Base URL):

  • http://localhost:3000/api/v1/products/:id → ganti :id dengan angka, contoh: http://localhost:3000/api/v1/products/1
  • http://localhost:3000/api/v1/customers/:id → ganti :id dengan angka, contoh: http://localhost:3000/api/v1/customers/2
  • http://localhost:3000/api/v1/transactions/:id → ganti :id dengan angka, contoh: http://localhost:3000/api/v1/transactions/1

Cara mengetahui ID yang tersedia:

  1. Jalankan GET semua data terlebih dahulu.
  2. Lihat nilai "id" pada data yang muncul.
  3. Gunakan nilai "id" tersebut untuk mengganti :id di URL.

Contoh: Jalankan GET http://localhost:3000/api/v1/products terlebih dahulu

[
    { "id": 1, "nama": "ASUS TUF A15", ... },
    { "id": 2, "nama": "iPhone 15", ... }
]

Maka untuk melihat detail produk pertama gunakan: http://localhost:3000/api/v1/products/1

🔄 Alur Request (Lifecycle)

Setiap request yang masuk ke TechStore API akan melewati siklus (lifecycle) berikut sebelum mengembalikan response ke pengguna:

Tahapan Validasi & Eksekusi Server

  1. 🖥️ Client (Postman / Browser): Mengirim HTTP Request (GET, POST, PUT, DELETE) beserta Header API Key ke endpoint tujuan.
  2. 🛣️ Router (routes/): Express.js menerima request dan mencocokkan URL dengan rute yang terdaftar.
  3. 🛡️ Middleware: Request dilewatkan ke logger.js, auth.js (Pengecekan API Key), dan validator.js (Pengecekan struktur JSON). Jika API Key salah, request langsung ditolak (Error 401).
  4. ⚙️ Controller (controllers/): Jika lolos validasi keamanan, request diproses oleh controller yang memuat logika bisnis utama (misal: pengecekan duplikat nama, menghitung subtotal transaksi, validasi ketersediaan stok).
  5. 🗄️ Database (MySQL): Controller mengeksekusi query SQL (via db.js) untuk membaca (SELECT) atau memanipulasi (INSERT/UPDATE/DELETE) data aktual di tabel.
  6. 📤 Response / Error Handler: Controller mengembalikan response berformat JSON ke Client secara rapi. Jika terjadi crash sintaks, ditangkap oleh errorHandler.js.

📮 Panduan Penggunaan CRUD & API Key di Postman

Langkah-Langkah Umum Testing API:

  1. Buka aplikasi Postman.
  2. Klik tombol + New Request (atau tanda tambah di tab atas).
  3. Pilih Method HTTP pada *dropdown* sebelah kiri (GET, POST, PUT, DELETE).
  4. Masukkan Endpoint URL Lengkap di kolom panjang (contoh: http://localhost:3000/api/v1/products).
  5. KEAMANAN (PENTING): Buka tab Headers, lalu pada kolom *Key* ketik x-api-key dan pada kolom *Value* ketik techstorekelompok5_2026. Pastikan kotaknya terceklis.
  6. Untuk metode POST dan PUT (Tambah/Update Data):
    • Pilih tab Body tepat di bawah kolom URL.
    • Pilih opsi raw.
    • Ubah dropdown *Text* di pojok kanan menjadi JSON.
    • Masukkan format/struktur kode JSON ke dalam kotak teks yang tersedia.
  7. Untuk metode GET dan DELETE: Tidak perlu mengisi tab Body. Biarkan kosong.
  8. Klik tombol biru Send.
  9. Hasil data (Response) akan langsung muncul di kotak bagian bawah aplikasi Postman.

📡 Dokumentasi API & Skenario Respon Sistem

Base URL Global: http://localhost:3000

🚫 Respon Kesalahan Umum (Global): API Key Ditolak / Unauthorized (401 & 403)

Sebagai catatan untuk pembaca: Karena semua endpoint di bawah ini (kecuali info server) berlindung di balik benteng keamanan API Key, maka jika pengguna sengaja atau lupa memasukkan parameter x-api-key yang benar di tab Headers Postman, sistem akan secara global membekukan permintaan tersebut dan mengembalikan respon error baku di bawah ini. Oleh karena itu, respon Unauthorized ini hanya akan dijabarkan di kotak ini saja dan tidak akan ditulis berulang-ulang di bawah setiap endpoint agar laporan lebih bersih dan ringkas.

{
    "status": "error",
    "message": "Akses ditolak. API Key tidak ditemukan pada Header 'x-api-key'." // (Jika Lupa/Kosong - Status 401)
}
{
    "status": "error",
    "message": "Akses ditolak. API Key yang Anda masukkan tidak valid." // (Jika Typo/Salah Kunci - Status 403)
}

🔒 Middleware

MiddlewareFungsi
LoggerMencatat setiap request yang masuk ke terminal.
Auth (API Key)Benteng utama yang memvalidasi otentikasi kunci sebelum eksekusi berlanjut.
ValidatorMengecek aturan logika data (misal: "Stok tidak boleh minus", "Harga wajib angka").
Error HandlerMencegah Node.js *Crash* dan mengembalikan error JSON yang bersahabat kepada kasir.
Not FoundMenangani route/URL sembarang yang ditebak orang.

📊 Status Code (HTTP Standar)

Status CodeKeterangan
200OK - Permintaan/Kalkulasi sukses.
201Created - Data baru sukses ditambahkan (Simpan Produk/Checkout).
400Bad Request - Format salah, ada duplikasi data, stok kurang, atau isi form kosong.
401Unauthorized - Klien tidak mengirimkan Headers x-api-key.
403Forbidden - Kunci x-api-key dimasukkan, namun salah / palsu.
404Not Found - ID Barang, Transaksi, Pelanggan, atau Endpoint URL tidak ditemukan.
500Internal Server Error - Kesalahan internal kode atau putus koneksi Database MySQL.

📝 Kesimpulan & Analisa Akhir

Sistem TechStore API terbukti sukses dikembangkan sebagai sebuah fondasi arsitektur backend Point of Sales (Kasir) bertaraf komprehensif, memanfaatkan kecepatan Node.js dan kehandalan basis data relasional MySQL.

Pengujian dan integrasi mendalam yang dilakukan memperlihatkan beberapa pencapaian arsitektur penting:

  • ✅ Integritas Database Skala Menengah: Sukses menghubungkan 4 tabel MySQL (Products, Customers, Transactions, Transaction Items) melalui konsep Foreign Key. Relasi ini mengikat setiap transaksi belanja ke profil pelanggan yang valid dan ke katalog barang yang konkret, menghindari resiko rekap transaksi siluman (transaksi tanpa pelanggan yang jelas).
  • ✅ Penanganan Error yang Sangat Kuat (*Robust Error Handling*): Sistem tidak hanya mampu melakukan simpan-tampil data (CRUD) secara lurus, tetapi juga dilengkapi dengan penanganan kondisi *edge-cases*. Aplikasi ini menolak nama produk ganda (mencegah duplikat), membatalkan pesanan kasir jika ID pelanggan tidak valid, dan dengan cerdas memblokir *checkout* jika permintaan beli melampaui sisa stok gudang (mengembalikan error 400 Bad Request).
  • ✅ Logika Bisnis Real-time: Sistem telah di-program untuk secara otomatis memotong stok barang di gudang inventaris secara instan setelah tabel transaksi berhasil mencatat *checkout*, menjaga sinkronisasi tanpa perlu intervensi manual pengelola (*zero manual updating*).
  • ✅ Keamanan Tertutup (*Zero Trust Architecture*): Penerapan Custom Middleware Authentication berhasil mengamankan semua operasi (*write/read/delete/update*). Akses sepenuhnya terkunci. Tanpa melampirkan kunci token x-api-key yang valid di komponen Headers, mustahil pihak luar bisa memanipulasi maupun sekadar melirik arus keuangan atau inventaris toko (*Unauthorized Protection*).
  • ✅ Intelijen Bisnis Lanjut (Laporan): Tidak sebatas menyimpan angka mati, API ini memiliki rute khusus (*Data Aggregation*) untuk mengekstrak data mentah menjadi wawasan bermakna (*Business Intelligence*), seperti menganalisis barang terlaris, menghitung grafik omzet harian otomatis, dan membunyikan alarm logik untuk barang yang akan segera habis.

Tindak Lanjut Pengembangan: Berdasarkan pengujian akhir dengan Postman, struktur backend ini dinyatakan matang, sangat aman dari malpraktik manipulasi *payload*, dan siap 100% untuk dikonsumsi oleh baris *code* aplikasi Frontend (misal: panel web Vue.js/React untuk operator toko, maupun aplikasi kasir Mobile/Android) sebagai otaknya.
