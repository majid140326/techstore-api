вњ… 5. Hasil Pengujian (Simulasi CRUD Postman)

Berikut adalah tampilan *real* dari proses Input dan Output JSON setiap kali kita melakukan aksi (CRUD) menggunakan aplikasi Postman.

A. Create Data (POST) - Tambah Produk Baru

POST
http://localhost:3000/api/v1/products

Langkah Penggunaan di Postman:

  1. Ubah metode di sebelah kiri URL menjadi POST.
  2. Ketik URL lengkap endpoint products.
  3. Di bawah URL, klik tab Body вћ” pilih raw вћ” ubah format ujung kanan ke JSON.
  4. Ketik kode JSON *Request* di bawah, lalu tekan tombol Send.
Request Body (Input)
{
  "nama": "MacBook Pro M3",
  "kategori": "Laptop",
  "harga": 25000000,
  "stok": 10
}
Response (Output)201 Created
{
  "message": "Produk berhasil ditambahkan",
  "insertId": 5
}

B. Read Data (GET) - Tampilkan Data Produk

GET
http://localhost:3000/api/v1/products/5

Langkah Penggunaan di Postman:

  1. Ubah metode menjadi GET.
  2. Ketik URL dengan menyertakan ID produk (contoh: 5) di akhir URL.
  3. Metode GET tidak membutuhkan Body. Langsung klik tombol Send.
Response (Output)200 OK
{
  "data": {
    "id": 5,
    "nama": "MacBook Pro M3",
    "kategori": "Laptop",
    "harga": 25000000,
    "stok": 10
  }
}

C. Update Data (PUT) - Ubah Harga & Stok

PUT
http://localhost:3000/api/v1/products/5

Langkah Penggunaan di Postman:

  1. Ubah metode menjadi PUT.
  2. URL harus mengandung ID produk target (contoh: 5).
  3. Masuk ke tab Body вћ” raw вћ” JSON.
  4. Ketik *Request* pembaruan (misal: harga diskon jadi 24 juta), klik Send.
Request Body (Input)
{
  "nama": "MacBook Pro M3",
  "kategori": "Laptop",
  "harga": 24000000,
  "stok": 15
}
Response (Output)200 OK
{
  "message": "Data produk ID 5 berhasil diupdate"
}

D. Delete Data (DELETE) - Hapus Produk

DELETE
http://localhost:3000/api/v1/products/5

Langkah Penggunaan di Postman:

  1. Ubah metode menjadi DELETE.
  2. Pastikan ID produk di URL sudah benar. Hati-hati karena data akan hilang.
  3. Kosongkan tab Body, lalu tekan Send.
Response (Output)200 OK
{
  "message": "Produk ID 5 berhasil dihapus dari database"
}

E. Proses Kasir (POST) - Membuat Transaksi Penjualan

POST
http://localhost:3000/api/v1/transactions

Langkah Penggunaan di Postman:

  1. Gunakan metode POST ke endpoint transactions.
  2. Pilih tab Body вћ” raw вћ” JSON.
  3. Kirim format ID customer pembeli, dan array items (berisi ID produk dan jumlah yang dibeli). Klik Send.
  4. *Catatan: Saat API ini dijalankan, stok produk otomatis akan berkurang di Database.
Request Body (Keranjang Belanja)
{
  "customer_id": 1,
  "items": [
    { "product_id": 1, "jumlah": 2 },
    { "product_id": 2, "jumlah": 1 }
  ]
}
Response (Struk Transaksi)201 Created
{
  "message": "Transaksi berhasil dibuat, stok produk telah terpotong",
  "data": {
    "transaction_id": 1,
    "total_harga": 39000000,
    "tanggal": "2026-06-11T08:45:41.000Z"
  }
}
