🖥️ TechStore API

REST API Sistem Kasir & Manajemen Inventori Toko HP dan Laptop

📚 Kelompok 5 🎓 Integrasi Sistem ⚡ Node.js + MySQL

📌 Pendahuluan

TechStore API adalah platform RESTful API backend yang dirancang khusus untuk mendukung operasional sistem kasir toko ritel HP dan Laptop. Menggunakan arsitektur MVC (Model-View-Controller) yang solid, API ini mampu mengotomatisasi pendataan stok barang, pengelolaan data pelanggan secara terpusat, validasi transaksi penjualan seketika, serta menghasilkan rekapitulasi data laporan bisnis yang akurat guna mendukung pengambilan keputusan manajemen.

🛠️ Teknologi yang Digunakan

🟢

Node.js

Runtime environment berbasis V8 Engine untuk mengeksekusi server JavaScript.

⚡

Express.js

Minimalist web framework untuk mengelola routing dan middleware HTTP server.

🗄️

MySQL

Sistem manajemen database relasional (RDBMS) yang diakses melalui package mysql2.

📮

Postman

Alat pengujian (API testing) endpoint secara intensif selama tahap pengembangan.

📁 Struktur Project

Implementasi kode memisahkan tanggung jawab (Separation of Concerns) dalam struktur folder yang clean:

TechStoreAPI/
├── src/
│ ├── config/
│ │ └── db.js # Koneksi database MySQL pool
│ ├── controllers/
│ │ ├── customers.controller.js # Logika bisnis manajemen data pelanggan
│ │ ├── laporan.controller.js # Pemrosesan query agregat untuk rekap laporan
│ │ ├── products.controller.js # Logika kontrol manipulasi stok dan produk
│ │ └── transactions.controller.js # Penanganan kalkulasi & invoice transaksi
│ ├── middleware/
│ │ ├── errorHandler.js # Interceptor error global (500 handler)
│ │ ├── logger.js # Logging request log otomatis (Method, URL, IP)
│ │ └── validator.js # Validasi struktur request body skema JSON
│ ├── routes/
│ │ ├── customers.routes.js # Definisi route entitas customer
│ │ ├── info.routes.js # Route meta-informasi aplikasi
│ │ ├── laporan.routes.js # Route endpoint khusus analisis statistika
│ │ ├── products.routes.js # Definisi route entitas produk
│ │ └── transactions.routes.js # Definisi route entitas transaksi
│ └── app.js # Inisialisasi Express app & integrasi middleware middleware
├── server.js # Entry point utama untuk menjalankan server di port 3000
├── package.json # Definisi dependency project & script runners
└── README.md # File dokumentasi utama repositori

🔒 Arsitektur Middleware Server

📝 Logger Middleware

Otomatis menyadap setiap paket data request HTTP yang masuk dan mencatat riwayat log waktu metode akses pada konsol server.

✅ Request Validator

Memfilter parameter body payload secara ketat. Menolak memproses request jika tipe data / field tidak sesuai kriteria.

❌ Global Error Handler

Menangkap semua jenis kegagalan crash sistem internal secara otomatis agar server tidak terputus (*down*) dan melempar status error rapi.

🔍 Not Found Interceptor

Mengamankan sistem dengan menangani request alamat URL liar di luar daftar endpoint resmi dan merespon dalam skema terstruktur.

📊 Standarisasi Respon Status HTTP Code

API ini mengadopsi standar kode status RESTful untuk mempermudah integrasi oleh tim Frontend / Mobile Engineer:

200
OK
201
Created
400
Bad Request
404
Not Found
500
Server Error

📝 Kesimpulan & Hasil Capaian Project

TechStore API berhasil diselesaikan oleh Kelompok 5 sebagai modul backend tangguh dengan pencapaian instrumen fitur terukur:


  • ✅ Penyelesaian modul operasi data CRUD terarah pada Products, Customers, dan Transactions.
  • ✅ Pengamanan integritas data server lewat rangkaian Middleware Logger, Validator, dan Error Handler.
  • ✅ Koneksi database relasional multi-tabel menggunakan MySQL terpusat melalui XAMPP local environment.
  • ✅ Penyediaan 5 variasi Laporan Intelijen Bisnis (Penjualan Global, Produk Terlaris, Kontribusi Pelanggan, Omset Harian, & Alert Restock).
  • ✅ Dokumentasi REST API transparan, lengkap, dan siap dipublikasikan via GitHub Pages/Readme dengan **Total 17 Endpoint Operasional**.
