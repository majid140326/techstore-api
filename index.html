📌 Pendahuluan

TechStore API adalah REST API yang dibangun menggunakan Node.js dan Express.js sebagai backend sistem kasir toko HP dan Laptop. API ini mengelola data produk, customer, transaksi penjualan, dan laporan secara lengkap dengan koneksi database MySQL (XAMPP). Seluruh proses di bawah ini mensimulasikan penggunaan melalui aplikasi Postman.

📦 Simulasi Siklus CRUD: Produk (Products)

Berikut adalah demonstrasi alur hidup sebuah entitas data. Kita akan membuat produk MacBook Pro M3, memanggil datanya, mengubah harganya, dan terakhir menghapusnya. Semua tersinkronisasi menggunakan ID Produk: 5.

Langkah 1: Tambah Produk Baru (Create)

POST
http://localhost:3000/api/v1/products

Cara Akses di Postman:

  1. Ubah tipe request menjadi POST di sebelah kiri kotak URL.
  2. Masukkan alamat endpoint: http://localhost:3000/api/v1/products
  3. Di bawah URL, klik tab Body ➔ pilih opsi raw ➔ ubah Text menjadi JSON di pojok kanan.
  4. Copy kode JSON request di bawah ini dan paste ke dalam kolom editor.
  5. Klik tombol biru Send.
Body (Raw - JSON)
{
  "nama": "MacBook Pro M3",
  "kategori": "Laptop",
  "harga": 25000000,
  "stok": 10
}
Response Status: 201 Created 
{
  "status": "success",
  "message": "Produk berhasil ditambahkan",
  "insertId": 5
}

Langkah 2: Tampilkan Data Produk (Read)

GET
http://localhost:3000/api/v1/products/5

Cara Akses di Postman:

  1. Buka tab baru (klik icon +).
  2. Ubah tipe request menjadi GET.
  3. Masukkan alamat endpoint ditambah ID produk di akhir URL: http://localhost:3000/api/v1/products/5
  4. Karena ini metode GET, Anda tidak perlu mengisi tab Body. Langsung klik tombol Send.
Response Status: 200 OK 
{
  "status": "success",
  "data": {
    "id": 5,
    "nama": "MacBook Pro M3",
    "kategori": "Laptop",
    "harga": 25000000,
    "stok": 10
  }
}

Langkah 3: Ubah Data Produk (Update)

PUT
http://localhost:3000/api/v1/products/5

Cara Akses di Postman:

  1. Ubah tipe request menjadi PUT.
  2. Masukkan alamat endpoint target (jangan lupa sertakan ID): http://localhost:3000/api/v1/products/5
  3. Masuk ke tab Body ➔ raw ➔ JSON.
  4. Masukkan kode JSON di bawah ini. Kita akan mengubah harga menjadi 24 Juta dan sisa stok menjadi 8.
  5. Klik tombol Send.
Body (Raw - JSON)
{
  "nama": "MacBook Pro M3",
  "kategori": "Laptop",
  "harga": 24000000,
  "stok": 8
}
Response Status: 200 OK 
{
  "status": "success",
  "message": "Data produk ID 5 berhasil diupdate"
}

Langkah 4: Hapus Data Produk (Delete)

DELETE
http://localhost:3000/api/v1/products/5

Cara Akses di Postman:

  1. Ubah tipe request menjadi DELETE.
  2. Masukkan URL dengan ID produk yang ingin dihapus: http://localhost:3000/api/v1/products/5
  3. Sama seperti GET, metode ini tidak membutuhkan Body. Langsung klik Send.
  4. *(Jika Anda mencoba melakukan metode GET lagi di URL yang sama setelah proses ini, status akan menjadi 404 Not Found).*
Response Status: 200 OK 
{
  "status": "success",
  "message": "Produk ID 5 berhasil dihapus dari database"
}

💰 Transaksi Penjualan (Kasir)

Contoh proses ketika pelanggan memproses pembayaran / checkout barang. Sistem akan memotong stok secara otomatis.

POST
http://localhost:3000/api/v1/transactions

Cara Akses di Postman:

  1. Ubah metode ke POST dan masukkan URL endpoint transaksi.
  2. Pilih tab Body ➔ raw ➔ JSON.
  3. Masukkan data ID customer dan list keranjang belanja (berupa array items dari product_id dan jumlah barang).
  4. Klik Send.
Body (Raw - JSON)
{
  "customer_id": 1,
  "items": [
    { "product_id": 1, "jumlah": 2 },
    { "product_id": 2, "jumlah": 1 }
  ]
}
Response Status: 201 Created 
{
  "status": "success",
  "message": "Transaksi berhasil dibuat, stok produk telah terpotong",
  "data": {
    "transaction_id": 1,
    "total_harga": 39000000,
    "tanggal": "2026-06-11T08:45:41.000Z"
  }
}
